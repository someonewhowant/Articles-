
В мире перегруженных веб-интерфейсов и тяжелых браузерных расширений иногда хочется иметь простой, быстрый и локальный инструмент под рукой. Pocket CLI — это минималистичный менеджер закладок, работающий прямо в терминале.

Главная особенность этого проекта — zero dependencies (нулевое количество сторонних зависимостей). Скрипт написан исключительно на встроенных модулях Node.js, что делает его легковесным и безопасным. Ниже мы разберем техническое устройство проекта и то, как начать им пользоваться.

# Создание Pocket CLI на Node.js: Пошаговое руководство

В этой статье мы разберем процесс создания мощного консольного менеджера закладок, вдохновленного сервисом Pocket. Мы реализовали интерактивный режим, систему тегов, поиск и красивый визуальный интерфейс, используя только встроенные модули Node.js.

## Архитектура приложения
Приложение построено вокруг нескольких ключевых концепций стандартной библиотеки Node.js:

CLI работает в двух режимах:

Single-command: Выполнение разовых команд (например, node pocket.js list).

Interactive (REPL): Полноценная консольная сессия, реализованная через встроенный модуль readline.

Кроссплатформенное взаимодействие с ОС: Модуль child_process и метод exec используются для открытия ссылок в браузере по умолчанию. Скрипт автоматически определяет операционную систему (process.platform) и подбирает нужную команду: start для Windows, open для macOS и xdg-open для Linux.

Хранение данных (File System): В качестве базы данных выступает обычный файл bookmarks.json. Модуль fs используется для синхронного чтения (readFileSync) и записи (writeFileSync). Это идеальный подход для локального CLI-инструмента, где не требуется сложная конкурентность.

Вместо использования популярных библиотек вроде Chalk, раскрашивание вывода в терминале реализовано через нативные ANSI escape-коды (например, \x1b[32m для зеленого цвета). Это позволяет создавать красивые таблицы и форматировать текст, не раздувая вес проекта
Приложение построено на базе трех основных китов:
1.  **Хранение**: Локальный JSON-файл для персистентности данных.
2.  **Интерфейс**: ANSI-последовательности для цветов и модуль `readline` для интерактивности.
3.  **Логика**: Обработка команд как через аргументы командной строки, так и внутри собственного REPL.

## Разбор ключевых фрагментов кода

### 1. Визуализация и цвета
Для создания приятного UI в терминале мы используем стандартные ANSI-коды.

```javascript
const colors = {
  reset: "\x1b[0m",
  bright: "\x1b[1m",
  fg: {
    green: "\x1b[32m",
    cyan: "\x1b[36m",
    yellow: "\x1b[33m",
    red: "\x1b[31m"
  }
};

const colorize = (color, text) => `${color}${text}${colors.reset}`;
```
**Объяснение**: Функция `colorize` оборачивает текст в управляющие последовательности, которые терминал интерпретирует как инструкции по смене цвета, и обязательно сбрасывает цвет в конце, чтобы не "покрасить" весь последующий вывод.

### 2. Управление данными (JSON DB)
Мы используем синхронные методы модуля `fs` для простоты реализации в рамках CLI-инструмента.

```javascript
const readDB = () => {
  try {
    const data = fs.readFileSync(DB_PATH, 'utf8');
    return JSON.parse(data);
  } catch (error) {
    return [];
  }
};

const writeDB = (data) => {
  fs.writeFileSync(DB_PATH, JSON.stringify(data, null, 2));
};
```
**Объяснение**: База данных представляет собой массив объектов в формате JSON. Мы используем `null, 2` в `JSON.stringify` для сохранения читаемости файла `bookmarks.json` (красивый отступ).

### 3. Интерактивный режим (REPL)
Одной из главных фишек стала возможность ввода команд без перезапуска скрипта.

```javascript
const startInteractiveMode = () => {
  const rl = readline.createInterface({
    input: process.stdin,
    output: process.stdout,
    prompt: colorize(colors.fg.green, 'pocket> ')
  });

  rl.prompt();

  rl.on('line', (line) => {
    // Парсинг строки с поддержкой кавычек
    const parts = line.trim().match(/(".*?"|[^"\s]+)+(?=\s*|\s*$)/g);
    // ... выполнение команды ...
    rl.prompt();
  });
};
```
**Объяснение**: Модуль `readline` позволяет слушать стандартный ввод (`process.stdin`). Регулярное выражение в `match` критически важно — оно позволяет вводить названия с пробелами в кавычках, например: `add http://url.com "Мой заголовок"`.

### 4. Умный формат дат
Вместо сухих ISO-штампов мы реализовали относительное время.

```javascript
const formatDate = (dateStr) => {
  const diffInSeconds = Math.floor((now - new Date(dateStr)) / 1000);
  if (diffInSeconds < 60) return 'just now';
  if (diffInSeconds < 3600) return `${Math.floor(diffInSeconds / 60)}m ago`;
  // ... и так далее
};
```

## Инструкция по использованию

### Основные команды
-   `add <url> [title] [--tags=t1,t2]` — Добавить новую ссылку.
-   `list` — Показать таблицу всех закладок.
-   `search <query>` — Поиск по всему (URL, заголовки, теги).
-   `show <id>` — Карточка с подробной информацией.
-   `open <id>` — Открыть ссылку в браузере.
-   `delete <id>` — Удалить запись.

### Запуск
Вы можете запустить приложение в интерактивном режиме:
```bash
node @pocket.js
```
Или выполнить одиночную команду:
```bash
node @pocket.js list
```
