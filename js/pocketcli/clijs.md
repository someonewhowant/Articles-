В мире перегруженных веб-интерфейсов и тяжелых браузерных расширений иногда хочется иметь простой, быстрый и локальный инструмент под рукой. Pocket CLI — это минималистичный менеджер закладок, работающий прямо в терминале.

Главная особенность этого проекта — zero dependencies (нулевое количество сторонних зависимостей). Скрипт написан исключительно на встроенных модулях Node.js, что делает его легковесным и безопасным. Ниже мы разберем техническое устройство проекта и то, как начать им пользоваться.

# Создание Pocket CLI на Node.js: Пошаговое руководство

В этой статье мы разберем процесс создания простого консольного менеджера закладок, вдохновленного сервисом Pocket. Для этого мы опишем его основные функции: интерактивный режим, систему тегов, поиск и визуальный интерфейс, используя только встроенные модули Node.js.

## Архитектура приложения
Приложение будет построенно вокруг нескольких ключевых концепций стандартной библиотеки Node.js:

1. **Режимы работы**: CLI работает как в режиме одиночных команд, так и в интерактивном режиме (REPL).
2. **Кроссплатформенность**: Автоматическое определение ОС для открытия ссылок.
3. **Хранение данных**: Использование JSON-файла и модуля `fs`.
4. **Визуализация**: Нативные ANSI-коды для оформления интерфейса.

## Подробный разбор кода

Разберем основные шаги реализации по частям, рассматривая каждую строку кода и её назначение.

### 1. Импорты и базовая настройка
Мы используем только стандартные модули Node.js. С их помощью мы будем работать с файловой системой, путями, системными командами и интерактивным вводом. Модуль fs отвечает за работу с файловой системой, path - за работу с путями, exec - за запуск системных команд, readline - за интерактивный ввод. 

```javascript
const fs = require('fs'); // Модуль для работы с файловой системой
const path = require('path'); // Модуль для работы с путями
const { exec } = require('child_process'); // Функция для запуска системных команд
const readline = require('readline'); // Модуль для интерактивного ввода

// Формируем путь к файлу БД в той же папке, где лежит скрипт
const DB_PATH = path.join(__dirname, 'bookmarks.json');
```

### 2. Цветовое оформление (ANSI Escape Codes)
Так как мы хотим держать проект без зависимостей, вместо библиотеки `chalk` мы используем встроенные коды терминала. Определим объкт `colors`, который будет содержать коды цветов, используемых в приложении. 

```javascript
const colors = {
  reset: "\x1b[0m", // Сброс всех стилей
  bright: "\x1b[1m", // Жирный текст
  fg: {
    green: "\x1b[32m", // Зеленый текст
    cyan: "\x1b[36m", // Циановый
    yellow: "\x1b[33m", // Желтый
    red: "\x1b[31m", // Красный
    blue: "\x1b[34m" // Синий
  }
};

// Функция-обертка для раскрашивания текста
const colorize = (color, text) => `${color}${text}${colors.reset}`;
```

### 3. Работа с базой данных (JSON)
При перезапуске мы очень не хотели бы, чтобы наши закладки терялись. Поэтому мы будем хранить их как минимум в JSON-файле. Для этой цели воспользуемся встроенными модулями `fs` и `path`. Определим путь к файлу JSON в той же папке, где лежит скрипт. Логика инициализации и чтения/записи данных. 

```javascript
// Если файла базы данных нет, создаем его с пустым массивом
if (!fs.existsSync(DB_PATH)) {
  fs.writeFileSync(DB_PATH, JSON.stringify([], null, 2));
}
```
### readDB - Чтение JSON-файла
Для чтения файла используем функцию `fs.readFileSync`. Она принимает путь к файлу и кодировку. Если файл не найден, то возвращается пустой массив. Ошибки выведем в консоль. Если в JSON-файле содержится некорректный JSON, то вернется пустой массив. 
```javascript
const readDB = () => {
  try {
    const data = fs.readFileSync(DB_PATH, 'utf8'); // Читаем файл
    return JSON.parse(data); // Парсим JSON в массив
  } catch (error) {
    console.error(colorize(colors.fg.red, 'Error reading database:'), error.message);
    return []; // В случае ошибки возвращаем пустой массив
  }
};

```

### writeDB - Запись JSON-файла
Для записи файла используем функцию `fs.writeFileSync`. Она принимает путь к файлу и данные. Если файл не найден, то он будет создан.

```javascript
const writeDB = (data) => {
  try {
    // Записываем данные с отступами в 2 пробела для читаемости
    fs.writeFileSync(DB_PATH, JSON.stringify(data, null, 2));
  } catch (error) {
    console.error(colorize(colors.fg.red, 'Error writing to database:'), error.message);
  }
};
```

### 4. Интерактивный режим (REPL)
Основу приложения составляет цикл чтения команд пользователя.
Тут мы воспользуемся встроенным модулем readline. Он предоставляет интерфейс для чтения данных построчно с любого потока, например, со стандартного ввода, что позволяет нам реализовать интерактивный режим работы приложения. Используем метод `createInterface` для создания интерфейса. В него передаются два аргумента: `input` и `output`. Оба указывают на стандартный поток ввода и вывода соответственно. В свойстве `prompt` мы передаем строку, которая будет выводиться перед вводом команды. Используем регулярное выражение для парсинга аргументов с учетом кавычек. 

```javascript
const startInteractiveMode = () => {
  const rl = readline.createInterface({
    input: process.stdin, // Откуда читаем (клавиатура)
    output: process.stdout, // Куда пишем (экран)
    prompt: colorize(colors.fg.green, 'pocket> ') // Приглашение к вводу
  });

  ```
StartInteractiveMode - это функция, которая запускает интерактивный режим работы приложения. Она создает интерфейс для чтения данных построчно с стандартного потока ввода и вывода. Использует регулярное выражение для парсинга аргументов с учетом кавычек.

```javascript
  rl.prompt(); // Показываем приглашение первый раз

  rl.on('line', (line) => {
    // Регулярное выражение для парсинга аргументов с учетом кавычек
    const parts = line.trim().match(/(".*?"|[^"\s]+)+(?=\s*|\s*$)/g);
    if (!parts) {
        rl.prompt();
        return;
    }
    const command = parts[0]; // Первое слово — команда
    // Удаляем лишние кавычки у аргументов
    const args = parts.slice(1).map(arg => arg.replace(/^"(.*)"$/, '$1'));
    
    executeCommand(command, args); // Выполняем команду
    rl.prompt(); // Снова показываем приглашение
  });
};
```

### 5. Открытие ссылок в браузере
На следующем шаге выполняем реализацию кроссплатформенной логики открытия URL. Для открытия ссылки в браузере используем модуль `child_process` и его метод `exec`. Определяем системную команду в зависимости от платформы и далее выполняем команду в операционной системе. 

```javascript
const open = (id) => {
  const db = readDB();
  const bookmark = db.find(b => b.id === parseInt(id));
  if (!bookmark) return console.error('Not found');

  // Определяем системную команду в зависимости от платформы
  const command = process.platform === 'win32' ? 'start' : process.platform === 'darwin' ? 'open' : 'xdg-open';
  
  // Выполняем команду в операционной системе
  exec(`${command} "${bookmark.url}"`, (err) => {
    if (err) console.error('Error opening URL:', err.message);
    else console.log('🚀 Opening...');
  });
};
```
Функция open - это функция, которая открывает ссылку в браузере. Она принимает id - идентификатор закладки. Читаем базу данных. Находим закладку. Определяем системную команду в зависимости от платформы. Выполняем команду в операционной системе. Если возникла ошибка, выводим её в консоль. Если закладка не найдена, выводим сообщение об ошибке.

### 6. Описание команд 
Добавление закладок осуществляется с помощью команды `add`. Она принимает в качестве аргументов название закладки и URL. Если URL не указан, то команда будет считать, что ему передано название закладки. Если URL не указан, то команда будет считать, что ему передано название закладки. Если URL не указан, то команда будет считать, что ему передано название закладки. 

```javascript
const add = (args) => {
  const db = readDB();
  const url = args[1] || `https://nodejs.org/en/docs/`; // Адрес, который сохраняем
  const title = args[0]; // Название закладки
  const newBookmark = {
    id: Date.now(), // Уникальный идентификатор
    title: title || `Bookmark ${db.length + 1}`, // Название закладки
    url: url // Адрес закладки
  };

  // Добавляем новую закладку в базу данных
  db.push(newBookmark);
  writeDB(db); // Записываем изменения в файл

  console.log(colorize(colors.fg.green, '✓'), `Added: "${newBookmark.title}"`);
};
```

За редактирование закладок отвечает команда `edit`. Она принимает в качестве аргументов id закладки, название закладки и URL. Если URL не указан, то команда будет считать, что ему передано название закладки. Если URL не указан, то команда будет считать, что ему передано название закладки. Если URL не указан, то команда будет считать, что ему передано название закладки.

```javascript
const edit = (id, title, url) => {
  const db = readDB();
  const bookmark = db.find(b => b.id === parseInt(id));
  if (!bookmark) return console.error('Not found');

  bookmark.title = title || bookmark.title;
  bookmark.url = url || bookmark.url;

  writeDB(db);
  console.log(colorize(colors.fg.green, '✓'), `Updated: "${bookmark.title}"`);
};
```

За удаление закладок отвечает команда `delete`. Она принимает в качестве аргументов id закладки. Если URL не указан, то команда будет считать, что ему передано название закладки. Если URL не указан, то команда будет считать, что ему передано название закладки. Если URL не указан, то команда будет считать, что ему передано название закладки.

```javascript
const deleteBookmark = (id) => {
  const db = readDB();
  const bookmark = db.find(b => b.id === parseInt(id));
  if (!bookmark) return console.error('Not found');

  const newDb = db.filter(b => b.id !== parseInt(id));
  writeDB(newDb);
  console.log(colorize(colors.fg.green, '✓'), `Deleted: "${bookmark.title}"`);
};
```

Список отображать будет команда `list`. Она принимает в качестве аргументов id закладки. Если URL не указан, то команда будет считать, что ему передано название закладки. Если URL не указан, то команда будет считать, что ему передано название закладки. Если URL не указан, то команда будет считать, что ему передано название закладки.

```javascript
const list = () => {
  const db = readDB();
  if (db.length === 0) {
    console.log('No bookmarks found. Add one with "add <title> <url>"');
    return;
  }
  
  // Выводим заголовок

};
```

### 7. Выполнение команд
Основной логикой приложения является обработка команд, которые вводит пользователь. Для этих целей мы реализовали функцию `executeCommand`. Основу функции составляет оператор `switch`, который анализирует первую команду из аргументов. В зависимости от значения команды выполняются соответствующие действия. Стоит отметить, что для каждой команды реализуется свой уникальный функционал. Кроме того, предусмотрен выход из приложения при вводе команд `exit` или `quit`, а также вывод справки при вводе команды `help`.


```javascript
const executeCommand = (command, args) => {
  switch (command) {
    case 'add':
      add(args); // Вызываем функцию add, передавая ей аргументы
      break;
    case 'list':
      list();
      break;
    case 'search':
      search(args[0]);
      break;
    case 'show':
      show(args[0]);
      break;
    case 'edit':
      edit(args[0], args[1], args[2]);
      break;
    case 'delete':
      deleteBookmark(args[0]);
      break;
    case 'open':
      open(args[0]);
      break;
    case 'help':
      help();
      break;
    case 'exit':
    case 'quit':
      process.exit(0);
    default:
      console.error('Unknown command:', command);
      help();
  }
};
```

## Инструкция по использованию

### Основные команды
-   `add <url> [title] [--tags=t1,t2]` — Добавить новую ссылку.
-   `list` — Показать таблицу всех закладок.
-   `search <query>` — Поиск по всему (URL, заголовки, теги).
-   `show <id>` — Карточка с подробной информацией.
-   `open <id>` — Открыть ссылку в браузере.
-   `delete <id>` — Удалить запись.

### Запуск
Для входа в интерактивный режим:
```bash
node pocket.js
```
Или выполнение одиночной команды:
```bash
node pocket.js list
```

Можем также сделать файл исполняемым:
```bash
chmod +x pocket.js
```
